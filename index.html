C语言核心语法(1)：第一个C程序

发表于 2025-12-09 更新于 2025-12-10 分类于 C语言 阅读次数： Valine：
本文字数： 844 阅读时长 ≈ 3 mins.

学习C语言的旅程，几乎都是从这个经典的“Hello, World!”程序起步的 —— 相信多数老师都会让大家先写下这段代码。

今天我们就借着它，拆解C语言的核心基础，迈出入门第一步。

为什么要这样写？每个部分都藏着关键逻辑

  1. int main()：程序的入口
    电脑运行程序时，必须明确“从哪里开始执行”——这是程序的核心起点。 C语言有个硬性规定：所有可执行程序都必须有且仅有一个main()函数，这就是程序的入口。

    这部分实际上有多种写法：

    1. int main(void): 无参数，返回int类型（标准写法）

    2. int main(int argc, char *argv[]): 带参数，返回int类型（应该是在工程中的常用写法）

    3. signed main(): 与int main()相同，返回int类型. 一般在算法竞赛中，在使用了#define int long long后，才会使用。

    4. void main(): 无返回值（最常见的非标准写法） (不建议使用)

    5. main(): 省略返回值类型（旧标准兼容写法） (不建议使用)

  2. { ... }：代码块

    main()后面的一对大括号{ }，划定了这个入口函数的代码块范围，意思是：所有在{ }里的代码，都将被视为main()函数的一部分，会被顺序执行。

    我们想让电脑做的所有操作（比如输出文字，或者数学运算），都必须写在这对大括号之间。

    并不只是函数后面需要放代码块，其他的一些关键字后也要放，比如if、for、while等，后面会详细介绍。

  3. printf("Hello, World!\n");：实现输出功能的核心指令

    我们的目标是在屏幕上打印“Hello, World!”这句话

    • 而printf()正是C语言用来格式化输出的工具（print=打印，f=format，即格式化）。

    • 括号里的"Hello, World!\n"是要输出的内容（双引号包裹的是“字符串”）。

    • 末尾的\n是换行符，能让输出后光标自动换行。

    • 最后必须加;，这是C语言的“语句结束符”，告诉电脑“这条指令到这就完了”。

  4. #include <stdio.h>：printf()的声明

    电脑不会凭空认识printf()函数，因为它不是C语言的“内置默认功能”，而是在C语言“标准输入输出库”（简称stdio库）里的工具。

    #include是C语言的预处理指令，作用相当于复制粘贴：在编译之前，#include <stdio.h>会把stdio库的所有内容（包含printf()的函数声明、使用规则）复制到当前程序中，告诉电脑：printf()是合法工具，该怎么调用它。

    如果少了这一行，电脑会不认识printf()，程序就会报错。

  5. return 0;：函数的“结束信号”

    回到main()函数开头的int：return 0;就是履行这个约定。

    当main()函数里的所有指令都执行完后，会向操作系统返回一个“0”，表示告诉系统：“这个程序正常执行完毕，没有出错”。

    实际上，从C++11开始，return 0;可以不用写，因为编译器会隐式插入 return 0;，等价于显式书写。（但是这并不代表其他函数也不用return，这只是因为 main 函数比较特殊而已）

数据库基础理论

数据库设计核心

E-R模型（实体-关系模型）

基本概念

  • 实体：能独立存在、有自己属性的对象，比如 “员工”“部门”“客户”“商品”。 每个实体都有 “唯一标识”（比如员工的 “工号”、商品的 “商品 ID”），后续会变成表的 “主键”。
  • 属性：实体拥有的具体信息，比如 “员工” 的姓名、工号、薪资、入职时间。 属性不能再拆成更小的 “有意义单位”（比如 “地址” 如果拆成 “省 + 市 + 区”，那 “省”“市”“区” 才是属性，原 “地址” 不是）。
  • 关系：不同实体之间的业务联系，比如 “员工属于部门”“学生选课程”，关系也有自己的 “类型”。| 常见关系类型（用 “一方”“多方” 理解，避免复杂术语）：
    • 一对一：两个实体 “一一对应”，比如 “员工” 和 “工牌”（一个员工只有一个工牌，一个工牌只属于一个员工）；
    • 一对多：一个实体对应多个另一个实体，比如 “部门” 和 “员工”（一个部门有多个员工，一个员工只属于一个部门）—— 最常见的关系；
    • 多对多：两个实体 “互相对应多个”，比如 “学生” 和 “课程”（一个学生选多门课，一门课有多个学生）。
